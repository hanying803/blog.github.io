hexo入门——初识

安装完成hexo之后，可以通过一下命令生成hexo的文件夹。

$ hexo init   <文件夹名>
$ cd <文件夹名>
$ npm install

初始化成功之后就会生成像下面的目录结果。

├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

_config.yml

站点的配置文件，大多数配置都可以通过这个文件进行。例如站点标题、描述等等。

package.json

应用插件数据，记录了需要安装的插件。

scaffolds

脚手架文件，也就是模板文件。当你创建新的文章时候，会利用对应的模板建立初识文件。像hexo new 文件名会按照post.md内容生成新的文章.

source

资源文件夹。站点的所有资源都存放在此处，包括markdown文章或者其他的资源。之后md文件会被渲染生成页面，其他格式资源会被复制到public文件中。

themes

主题文件夹。将搜集的主题放置在此处，通过配置可以使用主题进行站点装饰。
